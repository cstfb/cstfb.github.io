JAVA

volatile 的两个特性:可见行保证和禁止指令重排序；

public class Singleton {

    private static Singleton instance = null;
[]()
    private int v;
    private Singleton() {
        this.v = 3;
    }

    public static Singleton getInstance() {
        if (instance == null) { // 1. 第一次检查
            synchronized (Singleton.class) { // 2
                if (instance == null) { // 3. 第二次检查
                    instance = new Singleton(); // 4
                }
            }
        }
        return instance;
    }
}

大家都知道上面的代码存在代码重排序的问题，解决方案就是对instance变量加上volatile关键字，这样JVM会禁止在instance完成初始化之前进行引用赋值，这样解决了重排序的问题;(另外final关键字也能达到类似的效果)
